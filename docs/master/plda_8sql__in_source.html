19 @about
20 
21 Latent Dirichlet Allocation (LDA) is an interesting generative probabilistic
22 model for natural texts that has received a lot of attention in recent years.
23 The model is quite versatile, having found uses in problems like automated
24 topic discovery, collaborative filtering, and document classification.
25 
26 The LDA model posits that each document is associated with a mixture of various
27 topics (e.g. a document is related to Topic 1 with probability 0.7, and Topic 2 with
28 probability 0.3), and that each word in the document is attributable to one
29 of the document's topics. There is a (symmetric) Dirichlet prior with parameter
30 \f$ \alpha \f$ on each document's topic mixture. In addition, there is another
31 (symmateric) Dirichlet prior with parameter \f$ \eta \f$ on the distribution
32 of words for each topic. The following generative process then defines a distribution
33 over a corpus of documents. First sample, for each topic \f$ i \f$, a per-topic word distribution
34 \f$ \Phi_i \f$ from the Dirichlet(\f$\eta\f$) prior.
35 Then for each document:
36 -# Sample a document length N from a suitable distribution, say, Poisson.
37 -# Sample a topic mixture \f$ \theta \f$ for the document from the Dirichlet(\f$\alpha\f$) distribution.
38 -# For each of the N words:
39  -# Sample a topic \f$ z_n \f$ from the multinomial topic distribution \f$ \theta \f$.
40  -# Sample a word \f$ w_n \f$ from the multinomial word distribution \f$ \Phi_{z_n} \f$ associated with topic \f$ z_n \f$.
41 
42 In practice, only the words in each document are observable. The topic mixture of
43 each document and the topic for each word in each document are latent unobservable
44 variables that need to be inferred from the observables, and this is the problem
45 people refer to when they talk about the inference problem for LDA. Exact inference
46 is intractable, but several approximate inference algorithms for LDA have been
47 developed. The simple and effective Gibbs sampling algorithm described in
48 Griffiths and Steyvers [2] appears to be the current algorithm of choice. Our
49 parallel implementation of LDA comes from Wang et al [3], which is essentially
50 a straightforward parallelisation of the Gibbs sampling algorithm.
51 
52 See also http://code.google.com/p/plda/.
53 
54 @input
55 The \b corpus to be analyzed is expected to be of the following form:
56 <pre>{TABLE|VIEW} <em>datatable</em> (
57  <em>id</em> INTEGER,
58  <em>contents</em> INTEGER[],
59  ...
60 )</pre>
61 where \c id refers to the document ID, and \c contents is an integer array that specifies the words in the document using the index from
62 the dictionary. Words must be represented using positive numbers.
63 
64 The \b dictionary that indexes all the words found in the corpus is of the following form:
65 <pre>{TABLE|VIEW} <em>dicttable</em> (
66  <em>dict</em> TEXT[],
67  ...
68 )</pre>
69 
70 @usage
71 
72 - Topic inference is achieved through the following UDF
73  <pre>
74  SELECT \ref plda_run('<em>datatable</em>', '<em>dicttable</em>', '<em>modeltable</em>', '<em>outputdatatable</em>',
75  <em>numiter</em>, <em>numtopics</em>, <em>alpha</em>, <em>eta</em>);
76  </pre>
77  This function stores the resulting model in <tt><em>outputdatatable</em></tt>.
78 - Labelling of test documents using a learned LDA model is achieved using the following UDF
79  <pre>
80  SELECT \ref plda_label_test_documents('<em>testtable</em>', '<em>outputtable</em>', '<em>modeltable</em>', '<em>dicttable</em>',
81  <em>numtopics</em>, <em>alpha</em>, <em>eta</em>);
82  </pre>
83  This creates the following table with the assigned topic for each word in the test corpus.
84  <pre>
85  id | contents | topics
86  ----+-------------------------------+-----------------------------------
87  ...
88  </pre>

90 @implementation
91 The input format for the Parallel LDA module is different from that used by the
92 `lda' package for R. In the `lda' package, each document is represented by two
93 equal dimensional integer arrays. The first array represents the words that occur
94 in the document, and the second array captures the number of times each word in
95 the first array occurs in the document.
96 This representation appears to have a major weakness in that all the occurrences
97 of a word in a document must be assigned the same topic, which is clearly not
98 satisfactory. Further, at the time of writing, the main learning function in the
99 `lda' package actually does not work correctly when the occurrence counts for words
100 aren't one.
101 
102 There is a script called generateTestCases.cc that can be used to generate some
103 simple test documents to validate the correctness and effiency of the parallel
104 LDA implementation.

110 -# As a first step, we need to prepare a corpus and dictionary in the appropriate structure.
111  \code
112 sql> CREATE TABLE MADLIB_SCHEMA.plda_mydict ( dict text[] ) DISTRIBUTED RANDOMLY;
113 sql> INSERT into MADLIB_SCHEMA.plda_mydict values
114  ('{human,machine,interface,for,abc,computer,applications,a,survey,of,
115  user,opinion,system,response,time,the,eps,management,and,engineering,
116  testing,relation,perceived,to,error,generation,random,binary,order,tree,
117  intersection,graph,path,in,minor,IV,widths,well,quasi}');
118 
119 sql> CREATE TABLE MADLIB_SCHEMA.plda_mycorpus ( id int4, contents int4[] );
120 sql> INSERT INTO MADLIB_SCHEMA.plda_mycorpus VALUES
121  (1, '{1,2,3,4,5,6,7}'),
122  (2, '{8,9,10,11,12,10,6,13,14,15}'),
123  (3, '{16,17,11,3,18,13}'),
124  (4, '{13,19,1,13,20,21,10,17}'),
125  (5, '{22,10,11,23,14,15,24,25,18}'),
126  (6, '{16,26,10,27,28,29,30}'),
127  (7, '{16,31,32,10,33,34,30}'),
128  (8, '{32,35,36,37,10,30,19,38,39,29}'),
129  (9, '{32,35,8,9}') ;
130  \endcode
131 -# To perform inference, we call the plda_run() function with the appropriate parameters.
132  Here is an example.
133  \code
134 sql> select MADLIB_SCHEMA.plda_run('MADLIB_SCHEMA.plda_mycorpus', 'MADLIB_SCHEMA.plda_mydict',
135  'MADLIB_SCHEMA.plda_mymodel', 'MADLIB_SCHEMA.plda_corpus',
136  30,10,0.5,0.5);
137  \endcode
138  After a successful run of the plda_run() function, the most probable words associated
139  with each topic are displayed. Other results of the learning process can be obtained
140  by running the following commands. Here we assume the modeltable and outputdatatable
141  parameters to plda_run() are 'MADLIB_SCHEMA.plda_mymodel' and 'MADLIB_SCHEMA.plda_corpus' respectively.
142  -# The topic assignments for each document can be obtained as follows:
143  \code
144  sql> select id, (topics).topics from MADLIB_SCHEMA.plda_corpus;
145  \endcode
146  -# The topic distribution of each document can be obtained as follows:
147  \code
148  sql> select id, (topics).topic_d from MADLIB_SCHEMA.plda_corpus;
149  \endcode
150  -# The number of times each word was assigned to a given topic in the whole corpus can
151  be computed as follows:
152  \code
153  sql> select ss.i, MADLIB_SCHEMA.plda_word_topic_distrn(gcounts,$numtopics,ss.i)
154  from MADLIB_SCHEMA.plda_mymodel,
155  (select generate_series(1,$dictsize) i) as ss;
156  \endcode
157  where $numtopics is the number of topics used in the learning process, and
158  $dictsize is the size of the dictionary.
159  -# The total number of words assigned to each topic in the whole corpus can be computed
160  as follows:
161  \code
162  sql> select sum((topics).topic_d) topic_sums from MADLIB_SCHEMA.plda_corpus;
163  \endcode
164 -# To use a learned LDA model to label new documents, we can use the following commands:
165  \code
166 sql> select MADLIB_SCHEMA.plda_label_test_documents('MADLIB_SCHEMA.plda_mycorpus', 'MADLIB_SCHEMA.plda_testresult', 'MADLIB_SCHEMA.plda_mymodel', 'MADLIB_SCHEMA.plda_mydict', 10,0.5,0.5);
167 sql> select * from MADLIB_SCHEMA.plda_testresult;
168  \endcode

277 -- This function computes the topic assignments to words in a document given previously computed
278 -- statistics from the training corpus.
279 -- This function has not been rewritten in PL/Python because it is hard to handle structured types
280 -- like MADLIB_SCHEMA.plda_topics_t in PL/Python (this has to be done via conversion to text, which is not nice).
281 CREATE OR REPLACE FUNCTION
282 MADLIB_SCHEMA.plda_label_document(doc int4[], global_count int4[], topic_counts int4[], num_topics int4, dsize int4,
283  alpha float, eta float)
284 RETURNS MADLIB_SCHEMA.plda_topics_t AS $$
285 DECLARE
286  ret MADLIB_SCHEMA.plda_topics_t;
287 BEGIN
288  ret := MADLIB_SCHEMA.plda_random_topics(array_upper(doc,1), num_topics);
289  FOR i in 1..20 LOOP
290  ret := MADLIB_SCHEMA.plda_sample_new_topics(doc,(ret).topics,(ret).topic_d,global_count,topic_counts,num_topics,dsize,alpha,eta);
291  END LOOP;
292  RETURN ret;
293 END;
294 $$ LANGUAGE plpgsql;
