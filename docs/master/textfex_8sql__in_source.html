00017 @about
00018 The Feature Extraction module provides functionality for basic text-analysis
00019 tasks such as part-of-speech (POS) tagging and named-entity resolution.
00020 In addition to feature extraction, it also has a Viterbi implementation
00021 to get the best label sequence and the conditional probability
00022 \f$ \Pr( \text{best label sequence} \mid \text{Sentence}) \f$.
00023 
00024 At present, six feature types are implemented.
00025     - Edge Feature: transition feature that encodes the transition feature
00026       weight from current label to next label.
00027     - Start Feature: fired when the current token is the first token in a sentence.
00028     - End Feature: fired when the current token is the last token in a sentence.
00029     - Word Feature: fired when the current token is observed in the trained
00030       dictionary.
00031     - Unknown Feature: fired when the current token is not observed in the trained
00032       dictionary for at least certain times.
00033     - Regex Feature: fired when the current token can be matched by the regular
00034       expression.
00035 
00036 You can add your own feature type according to the training model.
00037 
00038 Instead of scanning every token in a sentence and extracting features for
00039 each token on the fly, we extract features for each distinct token and
00040 materialize it in the table.  When we call the Viterbi function to get the best
00041 label sequence, we only need a single lookup to get the feature weight.
00042 
00043 @usage
00044   - Use CRF PACKAGE [1] to generate the trained models. Basically CRF PACKAGE
00045     generates two model files 'feature' and 'crf'.  Write some simple scripts
00046     to convert the data in the model files to the data format required by in
00047     this module.
00048 
00049   - Load model from local drive to database
00050     <pre>SELECT madlib.load_crf_model(
00051          '<em>/path/to/data</em>');</pre>
00052 
00053   - Run feature extraction in batch mode or interactive mode
00054     <pre>SELECT madlib.text_feature_extraction(
00055          '<em>segmenttbl</em>',
00056          '<em>dictionary</em>',
00057          '<em>labeltbl</em>',

00059          '<em>featuretbl</em>',
00060          '<em>viterbi_mtbl</em>',
00061          '<em>viterbi_rtbl</em>');</pre>
00062 
00063   - Run the Viterbi function to get the best label sequence and the conditional
00064     probability \f$ p(top1_label_sequence|sentence) \f$.
00065     <pre>SELECT madlib.vcrf_label(
00066          '<em>segmenttbl</em>',
00067          '<em>viterbi_mtbl</em>',
00068          '<em>viterbi_rtbl</em>',
00069          '<em>labeltbl</em>',

00072 @literature

00083 /**
00084  * @brief This function extracts text features.
00085  *
00086  * This feature extraction function will produce two factor tables, "m table"
00087  * (\a viterbi_mtbl) and "r table" (\a viterbi_rtbl).  The \a viterbi_mtbl
00088  * table and \a viterbi_rtbl table are used to calculate the best label
00089  * sequence for each sentence.
00090  *
00091  * - <em>viterbi_mtbl</em> table
00092  * encodes the edge features which are solely dependent on upon current label and
00093  * previous y value. The m table has three columns which are prev_label, label,
00094  * and value respectively.
00095  * If the number of labels in \f$ n \f$, then the m factor table will \f$ n^2 \f$
00096  * rows.  Each row encodes the transition feature weight value from the previous label
00097  * to the current label.
00098  *
00099  * \a startFeature is considered as a special edge feature which is from the
00100  * beginning to the first token.  Likewise, \a endFeature can be considered
00101  * as a special edge feature which is from the last token to the very end.
00102  * So m table encodes the edgeFeature, startFeature, and endFeature.
00103  * If the total number of labels in the label space is 45 from 0 to 44,
00104  * then the m factor array is as follows:
00105  * <pre>
00106  *                  0  1  2  3  4  5...44
00107  * startFeature -1  a  a  a  a  a  a...a
00108  * edgeFeature   0  a  a  a  a  a  a...a
00109  * edgeFeature   1  a  a  a  a  a  a...a
00110  * ...
00111  * edgeFeature  44  a  a  a  a  a  a...a
00112  * endFeature   45  a  a  a  a  a  a...a</pre>
00113  *
00114  * - viterbi_r table
00115  * is related to specific tokens.  It encodes the single state features,
00116  * e.g., wordFeature, RegexFeature for all tokens.  The r table is represented
00117  * in the following way.
00118  * <pre>
00119  *        0  1  2  3  4...44
00120  * token1 a  a  a  a  a...a
00121  * token2 a  a  a  a  a...a</pre>
00122  *
00123  * @param segmenttbl Name of table containing all the testing sentences.
00124  * @param dictionary Name of table containing the dictionary.
00125  * @param labeltbl Name of table containing the the label space used in POS or other NLP tasks.
00126  * @param regextbl Name of table containing all the regular expressions to capture regex features.
00127  * @param viterbi_mtbl Name of table to store the m factors.
00128  * @param viterbi_rtbl Name of table to store the r factors.
00129  *
00130  */
