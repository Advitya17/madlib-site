00031 @anchor dls_about
00032 @about
00033 
00034 The linear systems module implements solution methods for systems of consistent
00035 linear equations.  Systems of linear equations take the form: 
00036 \f[
00037   Ax = b
00038 \f]
00039 
00040 where \f$x \in \mathbb{R}^{n}\f$, \f$A \in \mathbb{R}^{m \times n} \f$ and \f$b \in \mathbb{R}^{m}\f$. 
00041 We assume that there are no rows of \f$A\f$ where all elements are zero. 
00042 The algorithms implemented in this module can handle large dense
00043 linear systems. Currently, the algorithms implemented in this module
00044 solve the linear system by a direct decomposition. Hence, these methods are 
00045 known as <em>direct method</em>.

00121 @anchor dls_opt_params
00122 @par Optimizer Parameters
00123 
00124 For each optimizer, there are specific parameters that can be tuned
00125 for better performance.
00126 \par
00127 <DL class="arglist">
00128 <DT>algorithm (default: householderqr)</dT>
00129 <DD>
00130 
00131   There are several algorithms that can be classified as 'direct' methods
00132   of solving linear systems. MADlib dense linear system solvers provide 
00133   various algorithmic options for users.
00134   
00135   The following table provides a guideline on the choice of algorithm based 
00136   on conditions on the A matrix, speed of the algorithms and numerical stability.
00137     
00138        Algorithm            | Contitions on A  | Speed | Accuracy
00139        ----------------------------------------------------------
00140        householderqr        | None             |  ++   |  +
00141        partialpivlu         | Invertable       |  ++   |  +
00142        fullpivlu            | None             |  -    |  +++    
00143        colpivhouseholderqr  | None             |  +    |  ++
00144        fullpivhouseholderqr | None             |  -    |  +++
00145        llt                  | Pos. Definite    |  +++  |  +
00146        ldlt                 | Pos. or Neg Def  |  +++  |  ++
00147 
00148     For speed '++' is faster than '+', which is faster than '-'.
00149     For accuracy '+++' is better than '++'.
00150 
00151     More details about the individual algorithms can be found on the  <a href="http://eigen.tuxfamily.org/dox-devel/group__TutorialLinearAlgebra.html"> Eigen documentation</a>.  Eigen is an open source library for linear algebra.
00152 
00153 
00154 </DD>
00155 </DL>

00157 @anchor dls_output
00158 @par Output statistics 
00159 Output is stored in the <em>tbl_result</em> table.  
00160 <DL class="arglist">
00161 <DT>solution</dT>
00162 <DD>
00163 The solution is an array (of double precision) with the variables in the same
00164 order as that provided as input in the 'left_hand_side' column name of the
00165 'source_table'
00166 </DD>
00167 
00168 <DT>residual_norm</dT>
00169 <DD>
00170 Computes the scaled residual norm, defined as \f$ \frac{|Ax - b|}{|b|} \f$. 
00171 This value is an indication of the accuracy of the solution.
00172 </DD>
00173 
00174 <DT>iters</dT>
00175 <DD>`
00176 The number of iterations required by the algorithm (only applicable for 
00177  iterative algorithms). The output is NULL for
00178 'direct' methods. 
00179 </DD>
00180 
00181 </DL>
